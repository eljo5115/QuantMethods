Week 2 Monte Carlo Practice

I'd like for you to think about the game of Keno, and to submit a script in which you've used R to "play" several games of Keno. 

 Objectives

  1. Apply the sampling techniques from the video lessons to get R to play keno.
  2. Learn about writing a function in R.
  3. See a for loop in R.

 Activities

Part I: Play Keno

  • Open new script, immediately save it to the directory where you save your R cases.  Use the filename format R_Case_2_LAST_FIRST.R (for me, it would be "R_Case_2_Cookson_Tony.R"), and follow along with the explanation of the game below (and the *precise* instructions below):

    Keno: an explanation. How does Keno work?  There are 80 possible numbers (1, 2, 3,… 78, 79, 80), but each game only selects 20 of these 80 possible numbers.  Before seeing the 20 numbers that are randomly selected, a player chooses 8 numbers.  The cashier will charge you $1 per game you play, and will pay you according to the following payout table:

    Numbers that come up

    Payout

    8

    $15,000

    7

    $400

    6

    $50

    5

    $10

    4

    $2

    That is, you are paid $0 if 0, 1, 2, or 3 of your numbers come up, but $2 if 4 of your numbers come up, $10 if 5 of your numbers come up, and so on.

    Step-by-step instructions

    1. Use the sample command to simulate a game of Keno (that is, 20 draws of numbers between 1 and 80, without replacement).  This is similar to, but different, from rolling an 80-sided die 20 times. Around this spot Links to an external site. in the first video lesson, I covered a relevant line of code that you'll need to modify.
      • On line 2 of the script that you submit, please input the sample() command that accomplishes a (pseudo)random game of Keno.
    2. Now, "play" several games of Keno.
      • First, on line 6 of the script, store into an object called my_numbers a set of 8 numbers you'd like to play.
      • Second, on line 7, store an instance of the Keno game -- i.e., a vector of the 20 numbers that came up in the game -- into an object called numbers_on_the_board.
      • Third, on line 8, enter sum(my_numbers %in% numbers_on_the_board) into your script.  Literally just type this as is.  If you did lines 6 and 7 correctly, this code should compute the # of your numbers that "hit." And, you can compare to the payoff table to see what you've won.
      • Run lines 6, 7 and 8 several times to play Keno.  Re-run this code enough times to get an intuitive sense of how often you win (don't worry about systematically computing probabilities, yet).
      • Once you get an intuitive sense for if Keno is a good game to play to make money, think about amazing it would be to have a technique to run lines 6 through 8 a bunch of times without having to manually re-run it a bunch of times.... we'll get there soon.
    3. Set the seed (as we did in class with the rolling of dice) by typing set.seed(number) into line 5 of the script. Insert your favorite number into this command.
      • Run lines 5 through 8 several times to play Keno.  Re-run the code enough times to get an intuitive sense of what's going on here.
      • Do others in your breakout room get the same sequence of numbers?
      • Think: Knowing your favorite number, if I run your code on my computer, will I get the same answer as you?
    4. Insert comments using the # sign on lines 1 and 3 that describe generically what is going on in the lines below them.

    That's it.  The script is 8 lines, so far.

Part II: Play Keno: an example of writing a function.

For this next part of the case, I'm going to guide you through writing a function to play keno.  Functions in R are really useful, and just like in math, R functions have inputs and outputs.

To illustrate the syntax, the following code defines a function called plus that adds two inputs together and returns the output:

plus <- function(a,b){
  out <- a+b
  return(out)
}

Try it.  Run the above code to define the function (note that it shows up in the workspace), then run

plus(2,2)

in the console.

Functions can take vectors as an input, so we can write a plus function differently:

plus <- function(x){
  out <- x[1] + x[2]
  return(out)
}

Notice that x[1] takes the first element and x[2] takes the second.  To use this one, you need to give it a 2-element vector to add up.

numz <- c(2,2)
plus(numz)

But, it works the same.

To play keno using a function, we'll need to do something similar to the second plus function.  To know how you did, keno requires knowing what numbers you picked in advance.  Think of these as coming to the function as an 8-element input vector: my_numbers.

play_keno <- function(my_numbers){
  numbers_on_the_board <- sample(xxx, size = yyy, replace = FALSE)
  how_many_came_up <- sum(my_numbers %in% numbers_on_the_board)
return(how_many_came_up)
}

where xxx and yyy are numbers you figured out above in Part I.

Suppose you want to know your payout from the game.  You could look it up in the table above.  Here's a function that looks up your payout.

keno_payout <- function(how_many){
  samp_space   <- 0:8                               ## the nine outcomes that can come up
  payouts      <- c(0,0,0,0, 2, 10, 50, 400, 15000) ## the payouts for each of these 9 outcomes
  payout_table <- rbind(samp_space,payouts)         

  which_col <- payout_table["samp_space",] == how_many  ## What's going on here? 
  my_payout <- payout_table["payouts", which_col]
  return(my_payout)
}

Troubleshoot how the above function is working by understanding it "inside out."  Run the following code to see the logic behind the final product above.

samp_space <- 0:8 ## the nine outcomes that can come up
payouts <- c(0,0,0,0, 2, 10, 50, 400, 15000) ## the payouts for each of these 9 outcomes
payout_table <- rbind(samp_space,payouts) 

payout_table                 ## 1

payout_table["samp_space",]  ## 2

how_many <- 5  ## test case
payout_table["samp_space",] == how_many  ## 3
which_col <- payout_table["samp_space",] == how_many

payout_table["payouts", ]  ## 4

payout_table["payouts", which_col] ## 5

Repeat these in the script, and replace ## 1 through ## 5 with comments that explain what is going on.

With these functions, here's how you play keno.

my_numz <- c(5,10,15,20,1,2,3,4)    ## you choose these
how_many <- play_keno(my_numz)
my_money <- keno_payout(how_many)

Alternatively, you could put everything in one line, as follows:

my_money <- keno_payout(play_keno(c(5,10,15,20,1,2,3,4)))

What's next? Play Keno a bunch of times

The following code plays keno a bunch of times (technically, 10,000 times), and then stores it into an object called howd_i_do.

my_numz <- c(5, 15, 25, 35, 1, 2, 3, 4)
a_bunch <- 10000
howd_i_do <- rep(NA, a_bunch)

for(i in 1:a_bunch){
  howd_i_do[i] <- keno_payout(play_keno(my_numz))
}

Use the table command on the object howd_i_do to summarize the values that can come up, and their relative frequency (this is just a direct application of what I did in a simpler setting in this week's first video lesson Links to an external site.). 

Before you submit, a couple of things to note:

  1. You are likely to wonder: what's the expected payoff from keno? In the next unit, we'll discuss how to compute the expected value, but this should give you a picture of payouts from keno & how likely they are.  You may even anticipate where we're going and want to compute the expected payout now.
  2. We didn't do this case for pure entertainment. 
    • We'll extensively use for loops when simulating sampling distributions.
    • Simulation via for loops is a technique called Monte Carlo, which has a ton of practical applications.
    • Function writing in R is a nice way to hide gnarly code that you have to do a bunch of times.  Good to have early exposure.
  3. The other reason to do this case: Now, you have example code of a function and a for loop that you can adapt for other uses.
    • The best way to learn R is to see and adapt working code.  This is a good building block.  Save this script where you know to come back to it.

As with all first exposures, I don't expect everything to feel 100% comfortable right now.  However, when you see these things come up later, they'll be more familiar now that you've done the initial work.
